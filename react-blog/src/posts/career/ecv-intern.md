## 實習內容與經驗分享

我是【 2024 H2 eCloudture Cloud Seedling Program 雲端種子計畫】實習生，這期計畫時間是七月到九月，有關這三個月裡的實習內容我以週區分進行介紹：

### W1: 熟悉雲端
實習計畫的第一週主要是用公司的 eCloudture 線上學習平台去熟悉雲端。eCloudture 這個平台裡面有關於 AWS/Azure 服務很完整的線上課程影片，以及 Lab 環境讓我們在免費沙盒環境跟著步驟操作、熟悉雲端 console 介面，讓我們能夠在正式開始專案開發之前先打好基礎。

我自己認為 eCloudture 上的課程真的比 Udemy、AWS skill builder 等還要清楚與完整，完成課程後基本上就可以開始用 AWS 去進行開發了~

### W2~W4: 做中學

這部分其實每個實習生的實際工作內容都不大相同，以我們部門為例，我們的 mentor 很用心地設計了幾個 lab，讓我們透過實作熟悉 AWS 服務的運用以及 Well-Architected Framework 概念。那這段期間的學習方式是「邊做邊學」，透過實際操作來理解雲端架構的設計原則、網路連線串接、全端網頁架設、API、權限管理等等，為後續的專案開發打下扎實的基礎。

### W5~W11: 專案開發

在這近兩個月的時間內，我們實習生的任務是完成實習計畫的專案，並在結業式那天進行分享。我自己和另一位組員負責的是開發一個 WAF（Web Application Firewall）助手 MVP (Minimum Viable Product)，那這個專案的特色主要如下：

- **透過 IaC 即時跨帳號建置 WAF 資源**：使用基礎設施即程式碼（Infrastructure as Code）的方式，讓客戶能夠快速部署安全防護
- **常見網路攻擊 regex 規則包**：整合常見的攻擊模式，提供開箱即用的防護規則
- **GenAI 聊天機器人輔助建議配置**：利用生成式 AI 協助使用者進行 WAF 配置，降低技術門檻
- **簡化操作流程**：針對沒有資安或 AWS 相關背景的使用者，提供更友善的操作介面，解決原生 WAF 介面難以上手的問題

### W11~W13: 成果報告準備

在最後三週時間就是用來完成專案文件，還有準備全英文成果報告的內容。對我來說這其實是個全新的挑戰，因為它和過往在學校報告專案不同，這個成果報告面向的是企業內不同部門的主管，也就是包括人資、行銷等非技術背景的同仁，所以報告的重點不是著重在說明運用了什麼技術，而是要說服別人認可這項專案的價值。

## 個人成長與收穫

### 專業能力提升

在這次的實習計畫裡，我學會了很多專業技能，從 AWS 服務的運用、雲端系統開發、IaC 的使用，到網路知識的應用，這些都是我之前沒有深入接觸過的領域。透過實際的開發經驗，我不僅學會了如何使用 Terraform 來管理基礎設施，還學會了如何設計和建置企業級應用系統。

我們的 mentor 非常鼓勵我們主動學習。在這次實習以前，我其實沒有什麼軟體開發經驗，但在這三個月的實習期間，我學會了：

- **AWS 雲端開發實戰**：在三個月的密集開發讓我熟悉 AWS 在運算 (EC2/Lambda)、儲存 (S3/DynamoDB)、網路 (VPC/CloudFront) 及 AI (Bedrock) 等多項服務的應用。在面對開發瓶頸時，我也培養了閱讀 AWS 官方技術 Blog 與開發者文件來尋找解決方案的習慣，這不僅解決了當下的技術難題，更實質強化了我在解決方案的架構設計能力與系統排錯效率。
- **IaC 使用**：為了實現專案的自動化部署，我在一個月內從零開始學習 Terraform 這個基礎設施即程式碼的開發方式，並實際運用在專案。
- **網路知識應用**：學習 VPC、Gateway 等網路元件的設定，建置符合企業需求的應用系統
- **生成式 AI 應用開發**：運用 LangChain 框架進行開發，設計並實作具備對話記憶與檢索功能的 AI 聊天助手。

### 硬實力是你的起點，軟實力決定你的高度

大一大二期間只專注在累積專業知識，雖然知道軟實力是非常加分的能力，但對於它實際的影響力並沒有深刻的體會。在這三個月的職場洗禮後，我深刻體會到：

- **表達，把技術轉譯成價值**：在期末報告中有技術與非技術背景的主管，即便是技術主管也不一定了解我們的專案細節，所以說如何用通用的語言和表述方式讓大家看見專案價值，才能讓開發的努力被看見與重視。很謝謝我們的 mentor 安排我們練習報告給不同的導師、主管，讓我們在回饋中調整報告內容。最後我和組員用一個客戶職員的 daily routine 作為故事線來介紹這個專案。

    ![](images/intern/ecv/img3.png)

- **時間管理與目標聚焦**：明確的工作主線很重要。在短短不到兩個月的時間裡要完成一個半開放式的題目，我們很常遇到「新的想法不斷湧出」、「時間有限」的問題。雖然在亞馬遜的領導力準則中「Think Big」非常重要，但現實往往需要適度限縮目標並集中開發精力，如何實現 Pareto Principle（帕雷托法則）的核心概念 —— 以 20% 的活動取得 80% 的成果 —— 在這個時候就非常重要。

- **Growth Mindset**：公司的實習計畫很鼓勵實習生彼此交流，所以我們部門四位實習生都會找時間帶著電腦到交誼廳進行 coffee chat，分享彼此的開發進度、遇到的問題和新學的技術，這段經驗讓我體會到成長不是來自於證明自己什麼都懂，而來自於保持開放心態、在交流中不斷迭代思維。保持學習與分享的熱忱，比單打獨鬥更能突破個人的知識邊界

### 外語能力的提升

伊雲谷是個多元文化的企業，不只是拓展子公司到海外，光在台灣辦公室裡也有很多外籍同事，所以在整間企業裡多數溝通都是以英文為主。剛好很幸運的實習分組裡有兩位外籍生和我同部門，因此我有了很多練習口說的機會，讓我的英文表達從一開始連要用什麼單詞去表達都會卡住，到後來可以流暢地分享工作進度、討論技術問題，甚至是日常閒聊 XD。

### 職涯探索與方向確立

在參與實習之前，我對於雲端產業的了解只停留在雲端供應商和底下的客戶公司。進入到這個實習計畫後，因為實體和正職同仁在同一間辦公室上班、參與實習計畫準備給我們的企業文化講座、每天的導師時間，讓我認識了「雲端經銷商」這個企業定位以及工作內容，也更了解企業運作的規模與概念。

另外，這次實習計畫也讓我更確定自己對雲端產業與網路的熱情。起初是對於雲端與資安的好奇而加入，而在這段實習旅程中，我看見了資安更多的應用面向、企業真實需求以及職員的工作內容，這些收穫讓我清楚發現自己對於雲端開發有著濃厚的興趣，也為我未來的職涯方向提供了更明確的指引，真的是非常重要的收穫。

### 用紀錄驅動成長循環

在 ECV 裡每位實習生在打卡下班的時候都要記錄當天的工作或學習內容，而 mentor 也相當重視這件事，所以在這三個月裡，我們每天都會記錄自己的「Tasks」、「Lessons Learned」、「Unresolved Challenges」。

其實這些紀錄不只是讓我在回顧時能清楚看見成長軌跡，也幫助我更有方向地規劃下一個工作日的任務與時間分配。透過每天的整理與反思，我慢慢學會在行動前先思考、在完成後再檢視，讓學習與成長形成一個持續運轉的循環。而這個習慣也成為影響我後續學習與做事方式很重要的一部分！
