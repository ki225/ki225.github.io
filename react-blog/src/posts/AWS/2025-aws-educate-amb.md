## 前言

回顧整個 2025，從當上 AWS 校園大使、完成畢業專題、結束兩年的助教身分、踏入職場、順利推甄上研究所，充實的旅程讓我有了很大的轉變，也有了很多的收穫。但如果要說哪一段經歷帶來的成長最多，我會毫不猶豫地選擇加入 AWS Educate 校園大使計畫。

一開始報名，其實只是單純地想學雲端技術、拓展軟實力、認識對產業有熱情的人，沒想到這個「想多嘗試一點」的起點，最後成為我踏入雲端世界、持續走下去的起點。這一年下來，我不只是參與了許多活動和專案，也開始大量接觸 Serverless 架構、生成式 AI 技術、業界實戰應用，累積了許多寶貴的經驗與作品。而最讓我意外的是在一次次分享的過程中，我竟然也找到了熱愛的方向與價值。

那這篇文章會回顧我在第七屆 AWS 校園大使任期中參與的幾個重要里程碑，也希望藉由這樣的分享能讓更多人對於未來藍圖的規畫有了不一樣的想法～！

## 投入的活動

### Generative AI 工作坊 (3/7)
上任後不久就贏來了七屆技術大使的第一場大型活動，這場工作坊是為了讓對於之後黑客松有興趣的人可以更加瞭解 AWS 提供的 Gen AI 服務所準備的工作坊，當時因為熱衷於在 Pinterest 上找可愛圖片當頭貼，也非常依賴它以圖搜圖的功能，所以最後就決定以復刻 Pinterest 圖片搜尋系統為主題去設計這場工作坊，帶大家使用 Amazon Bedrock 多模態嵌入模型和 Amazon OpenSearch Serverless 做出了支援以文字或圖去搜圖的平台。

很感謝我的夥伴生了漂亮的包裝讓整個工作坊的成果更貼近復刻的樣子，還花了很多時間一起順教學簡報和流程，也非常感謝我的組長們給了許多回饋才能讓第一次的舞台有那麼好的回憶，最後總共有六十多位來自大專院校和企業的參與者來學習!

### 2025 AWS Educate Taiwan 雲端校園大使證照陪跑計畫

2025 的證照陪跑計畫進入第二屆，主要是希望透過推動這個活動讓更多人了解雲端產業的趨勢，並在有豐富資源和系統架構的學習環境中考取 AWS 證照，進而更好的進入相關產業。

陪跑計畫的內容非常多元，以學習資源就包括了小考題目、考古題整理、線上課程和工作坊(會在下面介紹到)以及 DC 社群疑難雜症解惑，另外我們也籌備了職涯分享、企業人才媒合會等活動讓學員可以更進一步了解這個產業，目的就是希望這是一個從技術到職涯、從考照到實務都能照顧到的計畫。

閱讀到這裡的你如果想了解更多當時計畫的資訊，可以參考[2025 AWS Educate Taiwan 雲端校園大使證照陪跑計畫](https://aws-educate-tw.notion.site/2025-AWS-Educate-Taiwan-1746bfee68178090a044fcb80214a5d5?pvs=74)的網頁，如果想參加大使近期的活動也歡迎關注粉絲專頁!

### 證照陪跑計畫工作坊 - AIF 證照課程

AIF 證照算是這兩年新出的證照，在我準備這場線上課程時網路上的考古題甚至只有 140 幾題，完整的考試範圍可以參考[官方的考試指南](https://d1.awsstatic.com/zh_CN/training-and-certification/docs-ai-practitioner/AWS-Certified-AI-Practitioner_Exam-Guide.pdf)。當時我們組有針對 Domain 2.3 會考的範圍去整理 AWS 生成式 AI 服務，並直播了線上課程去做介紹。

### 證照陪跑計畫工作坊 - The AI Tour：用 AI 洞察 Taylor Swift 演唱會的商業密碼 (3/29)

這場工作坊是為了推動學員去報考 AIF 這張證照所準備的實作課程，我們的核心學習目標是讓學員可以認識 Bedrock Flow 這項功能，去體會叫生成式 AI 模型去完成某一項任務時，有用 workflow 去拆解任務和沒有使用 workflow 差異，並搭配當時 Amplify 剛推出不久的 AI Kit 做出具有 Agent 調用工具功能的聊天介面。

順應當時泰勒斯演唱會的風潮，我們工作坊的實作成果是做出一個聊天介面，讓學員透過和介面互動去獲得 AI 根據資料庫所分析出來有關泰勒斯演唱會的商業資訊，當天有 75 位來自高中、大學和企業的人參與，最後也獲得 4.8/5 的滿意度!

### 中央資工 X 中大壢中 AWS CLOUD AI 主題式微課程 (5/17, 5/18)

在實際接觸業界之後，我深深相信校園與產業之間的銜接對學生來說非常關鍵，因為多數時候並不是學生能力不夠，而是資訊落差讓他們失去了探索更多可能的機會。

大二到大三期間我擔任系上「資訊與社會服務學習」課程的助教，這門課的核心理念，是希望透過培訓與實作，讓資工系的學生可以運用自己的專業能力回饋社會、做出影響。但我也常常在想，除了服務他人，這門課能不能也成為學生自身成長的跳板？讓他們帶得走一些技能，開啟對未來更多的想像。

過去這門課的每位負責人都會注入不同的創新元素，很幸運大三時我剛好有 AWS Educate 校園大使的資源，於是決定把 AWS 導入課程中。也很感謝教授的支持與信任，讓我們有機會與鄰近的中大壢中合作。
![alt text](images/AWS/2025-aws-educate-amb/img12.png)

經過三個月的籌備，我們規劃了一場為期兩天的 AWS 主題微課程，從基礎雲端知識開始講起，課程中穿插實作、討論與互動設計，讓 38 位高中同學在輕鬆的氛圍中接觸了資訊科技與雲端產業的第一步。

### 【Cake Meet 職涯探索】科技職涯體驗工作坊

原本以為第七屆的校園大使任期會在證照陪跑計畫畫下句點，沒想到在六月底收到 Cake 正準備與 AWS 合作的消息，預計舉辦一場針對社會新鮮人與轉職者的職涯探索活動，主題聚焦在如何進入雲端產業。很開心能受到邀請以校園大使的身分擔任實作工作坊的講師，帶領大家用 AWS Lambda、DynamoDB、S3 搭配 CloudFormation，部署出屬於自己的履歷網站。

那天來了超過 180 位來自各行各業、不同年齡層的參與者，從在職工程師到正在找方向的大學生都有。能夠透過這樣的場域把 AWS 技術分享給這麼多來自不同背景的人，並在活動結束後收到許多參與者正向的回饋，是一件非常有成就感的事!

### 2025 AWS Summit Taipei - Youth Tech Booth (7/29, 7/30)

這是我第一次參與偏行銷性質的活動，當天負責的是 AWS Summit Taipei 裡 Youth Tech Program 的展位。坦白說一開始真的蠻緊張的，畢竟過去我沒什麼在攤位上長時間和人互動、主動介紹技術或分享經驗的機會，而且當天來參觀的與會者背景非常多元，從大學生、政府單位官員，到企業主管都有，完全無法預測會被問到什麼問題。

## Dev Team - Open Source Side Project

### TPET，從解決團隊問題到走向社群
[TPET](https://tpet.aws-educate.tw/) 是 AWS Educate Dev Team 為了提升大使在寄活動信件效率所開發的產品，更多有關於 Dev Team 成立的時空背景可以參考 Shiun 的[這篇筆記](https://shiun.me/blog/my-growth-journey-as-an-aws-educate-cloud-ambassador/)。

雖然我不是創始元老，但算下來我也加入 Dev Team 快滿一年了，在過程中我們不斷朝著推出「社群版 TPET」的目標前進，並在七屆的任期內進行 TPET 介面大改版、資料庫轉用無伺服器的 Aurora Serverless v2 來減省成本與實現分頁機制，近期也開始著手 Alert 系統、人員管理系統等功能，期待能在未來一年內將 TPET 這個產品的功能完善，並釋出社群版本。


### 考取證照

我的第一張 AWS 證照是在 2024 年的暑假考到的 CPE，那時的準備是跟著第一屆陪跑計畫的課程。而在擔任大使的這一年裡，我則是在年初挑戰 AIF，接著在下半年考取了 SAA。蠻幸運的是，當時 SAA 的考試費用剛好有計畫幫忙申請到免費券！（雖然未來不一定會有 XD 像我當時考 AIF 也是半價考的）

如果你正考慮加入大使計畫、進入雲端產業，但沒有任何 AWS 相關的背景且不知道怎麼下手，其實蠻推薦可以先從 CPE 開始準備。這張證照比較像是 AWS 的 overview，對於想快速了解雲端到底能做什麼、有哪些服務很有幫助。考完後無論是接下來自己做專案還是聽別人分享架構設計，都會比較能跟上節奏。如果你的目標是使用 AWS 上的生成式 AI 服務，那我會建議 CPE 考完後接著挑戰 AIF，甚至可以一起準備，因為 AIF 裡面也會考到一些 CPE 的核心服務使用，有了 CPE 的基礎再準備 AIF 會比較有延續性。

當考完 CPE 後期時就可以開始嘗試開發 AWS 專案了，如果未來有興趣在 AWS 產業裡面做開發，我覺得考 SAA 會是一個很好的加分項，不過一開始準備會稍微吃力一些，我還記得那時誤打誤撞寫到 SAA 題目，結果錯的一蹋糊塗QQ  但後來因為做的專案多了，實戰經驗累積起來，反而考前只花一週時間刷題就順利通過了。我自己對於準備 SAA，我的建議是多去看、去聽別人怎麼設計架構，特別是電商這種重視高可用性的場景，會跟 SAA 的出題方向很貼近。多理解一些實際案例，再搭配自己動手實作部署配置，就能把 SAA 掌握得差不多了。不過畢竟 AWS 的服務很多，不可能每個都碰過，小細節還是得靠刷題補強，久了自然會抓到雲端平台的學習節奏～


## 收穫

- 熟悉雲端

    擔任 AWS 校園大使讓我以一個非常實戰導向的方式學習雲端技術。無論是透過準備 workshop、與其他技術大使的討論交流，還是參與 Dev Team 的專案開發，每一個過程都幫助我在短時間內打下紮實的基礎。

- 社群與個人品牌

    加入大使後，我第一次開始經營 LinkedIn，並在上面分享自己參與計畫的經歷。也因為大使的身分與舞台，讓我開始累積自己的個人品牌，有越來越多機會被看見，甚至接觸到與企業合作的可能性。

- 實習機會

    AWS 在台灣有很多企業合作夥伴。雖然大三那年因為忙著課業，沒特別走刷題路線去申請 SWE 實習，但憑著自己做的 AWS 專案、以及考取的證照，還是順利找到了一個很喜歡的實習機會!

## 心得

在參與大使計畫的這一年裡，除了對雲端產業有了更深入的認識，另一個很大的收穫就是體會到「社群的力量」。

當你踏進一個正向、有熱情的社群，無論是知識、人脈，還是軟實力與機會，都會以超乎預期的速度成長。很感謝大使計畫的一路上遇到許多願意分享、互相幫忙的夥伴，讓我在學習的路上獲得了很多能量，也成功找到自己在資訊領域上努力的目標與方向。接下來我也會以第八屆 AWS 校園大使的身分繼續參與計畫，期待能把這一年累積的經驗轉化成下一個人學習路上的一盞燈，讓更多對這個領域有興趣的人能帶著熱血走進雲端的世界 (ˊ v ˋ )
